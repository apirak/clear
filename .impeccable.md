# Impeccable Design Context

## Users

**Teachers** (primary) — answering student questions throughout the day. Context: busy, managing many questions, need efficient triage. Job: review, answer, and close question tickets quickly.

**Students** (secondary) — asking questions when stuck. Context: frustrated or curious, need clear answers. Job: ask a question, get a helpful response.

Thai university education. Bilingual TH/EN. Used on desktop during office hours, possibly mobile between classes.

## Brand Personality

**Modern. Academic. Professional.**

Premium edtech tool — not a consumer chat app, not a sterile LMS. Feels like a well-designed research tool that respects your time. Polished but approachable. Scholarly without being stiff.

**Emotional goals:** Confidence (teachers feel in control), Clarity (no ambiguity in status or action), Trust (reliable, nothing surprising).

## Aesthetic Direction

**Visual tone:** Minimal chrome, strong typography, generous whitespace. Intercom/Zendesk inbox patterns — efficient triage with clear visual hierarchy and status indicators.

**References:**
- Intercom/Zendesk — inbox triage patterns, clear status, professional helpdesk feel
- Linear/Notion — minimal, keyboard-driven, premium whitespace usage

**Anti-references:**
- Consumer chat apps (WhatsApp, LINE) — too casual, lacks academic authority
- Traditional LMS (Moodle, Blackboard) — cluttered, dated, painful to use

**Theme:** Light-first with dark mode support. Teal/aqua palette (sea-ink, lagoon, palm) — NOT the indigo/emerald from DESIGN.md. The current `styles.css` color direction is closer to intent.

**Typography:** Migrate to Sarabun for Thai/Latin rendering. Display and body scale from DESIGN.md typography tokens still apply.

## Design Principles

1. **Triage-first** — Teachers scan and act fast. Status, priority, and unread state must be instantly scannable. Every pixel of the inbox layout earns its place.

2. **Academic authority** — The interface should feel like a tool built for educators, not a consumer product. Restrained color, strong type, minimal decoration.

3. **Calm confidence** — No urgency mechanics or attention-grabbing animations. Teachers are already busy; the UI should feel stable and predictable.

4. **Bilingual clarity** — Thai and English coexist. Layout must handle Thai script (taller line height, no descender clipping). Font choice (Sarabun) is a design decision, not just i18n.

5. **Progressive density** — Default view is spacious. Power users can reveal more information density through panel toggles and collapsible sections. Never cram.

## Technical Notes

- Tailwind CSS v4 (CSS-first config in `src/styles.css`)
- Shadcn/ui as component foundation
- Three-panel inbox layout (240px / 320px / flex / 300px)
- Dark mode via `.dark` class, persisted localStorage
- WCAG AA compliance target
- Sarabun font (Google Fonts) for Thai/Latin
